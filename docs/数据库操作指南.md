## 1. 进入云开发控制台

1. 打开 **微信开发者工具**
2. 点击工具栏的 **云开发** 按钮
3. 选择对应的云环境
4. 点击左侧菜单 **数据库**

## 2. 数据库集合说明

### 2.1 users 集合（用户表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | String | 记录ID（自动生成） |
| `_openid` | String | 用户微信OpenID |
| `role` | String | 角色：`student`（学生）/ `employer`（甲方） |
| `realName` | String | 真实姓名 |
| `certStatus` | String | 认证状态：`unverified` / `pending` / `approved` / `rejected` |
| `certImages` | Array | 认证图片URL数组 |
| `createTime` | Date | 创建时间 |

### 2.2 jobs 集合（需求表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | String | 记录ID |
| `_openid` | String | 发布者OpenID |
| `title` | String | 需求标题 |
| `description` | String | 需求描述 |
| `salary` | String | 薪资范围 |
| `contact` | String | 联系方式 |
| `skills` | Array | 技能要求 |
| `status` | String | 状态：`active` / `closed` |
| `createTime` | Date | 发布时间 |

### 2.3 comments 集合（评论表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | String | 记录ID |
| `_openid` | String | 评论者OpenID |
| `jobId` | String | 关联的需求ID |
| `content` | String | 评论内容 |
| `status` | String | 状态：`pending` / `approved` / `rejected` |
| `createTime` | Date | 评论时间 |

## 3. 用户管理操作

### 3.1 修改用户角色（学生 → 甲方）

**场景**：用户注册时选错了角色，需要从学生改为甲方

**操作步骤**：

1. 进入 **数据库** → 选择 **users** 集合
2. 找到目标用户记录（可通过 `realName` 或 `_openid` 搜索）
3. 点击该记录进入编辑模式
4. 修改字段：
   ```json
   {
     "role": "employer"
   }
   ```
5. **重要**：同时将认证状态重置为未认证
   ```json
   {
     "role": "employer",
     "certStatus": "unverified",
     "certImages": []
   }
   ```
6. 点击 **保存**

**原因说明**：学生和甲方的认证材料要求不同，切换角色后需重新提交认证。

### 3.2 修改用户角色（甲方 → 学生）

**操作步骤**：

1. 进入 **数据库** → 选择 **users** 集合
2. 找到目标用户记录
3. 修改字段：
   ```json
   {
     "role": "student",
     "certStatus": "unverified",
     "certImages": []
   }
   ```
4. 点击 **保存**

### 3.3 查询特定角色的用户

**查询所有学生**：
```json
{
  "role": "student"
}
```

**查询所有甲方**：
```json
{
  "role": "employer"
}
```

### 3.4 删除用户

**警告**：删除用户会导致其发布的需求和评论成为孤立数据

1. 找到目标用户记录
2. 点击 **删除** 按钮
3. 确认删除

## 4. 认证审核操作

### 4.1 查看待审核用户

1. 进入 **数据库** → 选择 **users** 集合
2. 点击 **查找** / **高级筛选**
3. 输入查询条件：
   ```json
   {
     "certStatus": "pending"
   }
   ```
4. 点击 **查询**

### 4.2 审核通过

1. 找到待审核用户记录
2. 查看 `certImages` 字段中的图片URL
3. 在浏览器中打开图片进行核验
4. 核验通过后，修改字段：
   ```json
   {
     "certStatus": "approved"
   }
   ```
5. 点击 **保存**

### 4.3 审核拒绝

1. 找到待审核用户记录
2. 修改字段：
   ```json
   {
     "certStatus": "rejected"
   }
   ```
3. 点击 **保存**

**可选**：添加拒绝原因字段
```json
{
  "certStatus": "rejected",
  "rejectReason": "照片模糊，请重新上传"
}
```

### 4.4 重置用户认证状态

**场景**：用户认证被拒绝后需要重新提交

```json
{
  "certStatus": "unverified",
  "certImages": []
}
```

## 7. 批量操作技巧

### 7.1 使用云函数批量更新

如需批量操作，可在云函数中编写脚本：

```javascript
// 示例：批量通过所有待审核用户
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  return await db.collection('users')
    .where({ certStatus: 'pending' })
    .update({
      data: { certStatus: 'approved' }
    })
}
```

### 7.2 数据导出

1. 在数据库集合页面
2. 点击 **导出** 按钮
3. 选择导出格式（JSON/CSV）
4. 下载数据备份

## 注意事项

1. **数据备份**：重要操作前建议先导出数据备份
2. **权限控制**：确保只有管理员能访问云开发控制台
3. **操作日志**：建议记录每次审核操作的时间和操作人
4. **用户通知**：审核结果变更后，用户下次打开小程序时会自动更新状态
